## Polish commit lessons

The 6 phases all shipped with passing unit tests + curl-based live-test, but the playwright sweep caught:

1. **Project scope bug** — `loadG9Panels` had `if (role === 'project') return` *before* the org-only-widget hide branch, so toggling from org → project left the rework + mix-leaderboard cards visible. Unit tests don't exercise role transitions in the JS state machine.
2. **Engineer KPI strip wrap** — reused `.project-hero-grid` (4 cols) but ships 5 tiles; the 5th wrapped to its own row on desktop. Visual-only — no test would catch this.
3. **Mobile DORA overflow** — pre-existing v0.7.0 issue, not G10. But discovered during G10 sweep, so fixed in the polish commit.
4. **gofmt drift** — committed earlier without `gofmt -w`. Caught by `gofmt -l` during release readiness check.

Lesson: even with TDD + curl live-test, **a browser-level visual sweep is the only thing that catches scope-transition bugs and CSS-layout regressions**. Made it part of the release-readiness checklist for v0.9.0+.
