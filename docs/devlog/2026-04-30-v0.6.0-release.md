# 2026-04-30 — v0.6.0 release

## What shipped

Single feature release: G8 Intent Preservation.

**Features:**
- 3 new layer-4 events: `intent.extracted`, `decision.point`, `agent.reasoning`
- `dandori incident-report --run <id>` / `--task <key>` command (markdown post-mortem)
- Jira completion comment extension: `### Intent` + `### Key Decisions` sections appended when intent events exist
- Env gate: `DANDORI_INTENT_DISABLED=1`

**Fixes:**
- Redact regex no longer false-positives on prose like "password hashing" / "reset the user token". Now requires explicit `=` / `:` assignment delimiter or quoted JSON form.

**Storage impact:**
- Zero schema migration. Reuses `events` table.
- ~30% size increase per run (1.5–2 KB → 2.5–3 KB). 100-eng team adds <4 GB/year.

## Why one consolidated phase plan, then a single release

6 phases (P1 parser, P2 decisions, P3 spec linkage, P4 Jira comment, P5 incident-report, P6 tests+docs) implemented sequentially with parallel P4/P5 dispatch. ~110 unit/integration tests added across 5 packages. Zero regressions.

Single PR-equivalent commit (no upstream PR cycle — workspace is owner's repo).
